# Port Manager 快速启动指南

## 📦 发布包内容

```
release/
├── PortManager.command    ← Mac系统双击运行
├── PortManager.bat        ← Windows系统双击运行
├── port-manager-web.jar   ← 应用程序主文件
└── README.txt             ← 详细说明文档
```

---

## 🚀 一键启动

### Mac用户
```bash
双击 PortManager.command 即可启动
```

**首次运行提示**:
- 如果Mac提示"无法打开"，右键点击文件选择"打开"
- 或在终端中运行: `./PortManager.command`

### Windows用户
```bash
双击 PortManager.bat 即可启动
```

**首次运行提示**:
- 如果Windows提示安全警告，点击"仍要运行"

---

## ✨ 启动后自动完成

1. ✅ 检测Java环境
2. ✅ 检查端口占用
3. ✅ 启动Port Manager应用
4. ✅ 自动打开浏览器到管理页面

**访问地址**: http://localhost:9527

---

## ⚙️ 系统要求

### 必需环境
- **Java**: 11或更高版本
- **端口**: 9527端口可用

### 安装Java (如果未安装)

#### Mac
```bash
brew install openjdk@11
```

#### Windows
访问 https://adoptium.net/ 下载安装包

## 🎯 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 实时监控 | 自动扫描所有端口占用情况 |
| 🎛️ 多维筛选 | 按端口类型、进程类型、协议等筛选 |
| ❌ 一键关闭 | 单个或批量关闭进程 |
| 🛑 永久停止 | 永久停止系统服务（不自动重启） |
| 📊 智能识别 | 自动识别开发进程、数据库、Web服务器 |
| 🌓 主题切换 | 支持浅色/深色主题 |

## 💡 使用技巧

### 1. 筛选特定端口
```
使用顶部筛选器 → 选择端口类型/进程类型 → 快速定位
```

### 2. 批量关闭端口
```
勾选要关闭的端口 → 点击"关闭选中" → 确认关闭
```

### 3. 永久停止服务
```
点击"关闭"按钮 → 选择"永久停止" → 确定
```

### 4. 关闭数据库服务
```
筛选"进程类型: DATABASE" → 批量选择 → 永久停止
```

## 🛠️ 常见问题解决

### ❓ 提示"未检测到Java环境"
```
→ 安装Java 11或更高版本
→ Mac: brew install openjdk@11
→ Windows: https://adoptium.net/
```

### ❓ 提示"端口9527已被占用"
```
→ 启动脚本会询问是否关闭占用进程
→ 输入 y 自动关闭
```

### ❓ 浏览器没有自动打开
```
→ 手动访问 http://localhost:9527
```

### ❓ 关闭进程后又重新出现
```
→ 这是系统管理的服务（如MySQL、Redis）
→ 使用"永久停止"选项而不是"临时关闭"
```

### ❓ Mac提示"无法打开未验证的开发者"
```
→ 方法1: 右键点击 → 选择"打开"
→ 方法2: 系统偏好设置 → 安全性与隐私 → 点击"仍要打开"
→ 方法3: 终端运行 chmod +x PortManager.command
```

## 📝 停止应用

### Mac
```
按 Ctrl+C 或直接关闭终端窗口
```

### Windows
```
在命令窗口中按任意键或直接关闭窗口
```

应用会自动清理并停止所有相关进程。
